# User Stories

> 형식: Epic-Based / Task 수준 / Acceptance Criteria: 혼합 (Given-When-Then + 체크리스트)
> 페르소나: 고객(김민준), 관리자(이수진)

---

## Epic 1: 고객 인증 및 세션

### US-C-01: 테이블 초기 설정 (관리자 수행)
**As a** 매장 관리자(이수진)로서,  
**I want** 태블릿에 매장 식별자, 테이블 번호, 비밀번호를 1회 입력하고 저장하기를,  
**So that** 고객이 별도 로그인 없이 해당 테이블에서 바로 주문할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 관리자가 초기 설정 화면에 접근했을 때
When 매장 식별자, 테이블 번호, 비밀번호를 입력하고 저장 버튼을 누르면
Then 설정 정보가 로컬 스토리지에 저장되고 자동 로그인이 활성화된다
```

**추가 조건 (체크리스트)**:
- [ ] 매장 식별자, 테이블 번호, 비밀번호 모두 필수 입력
- [ ] 저장 성공 시 메뉴 화면으로 자동 이동
- [ ] 잘못된 자격증명 입력 시 에러 메시지 표시

---

### US-C-02: 자동 로그인
**As a** 고객(김민준)으로서,  
**I want** 테이블 브라우저를 열면 자동으로 로그인되기를,  
**So that** 별도 절차 없이 즉시 메뉴를 볼 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 테이블에 초기 설정이 완료된 상태에서
When 고객이 브라우저를 열거나 페이지를 새로고침하면
Then 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다
```

**추가 조건 (체크리스트)**:
- [ ] 새로고침 후에도 로그인 상태 유지
- [ ] 초기 설정이 없는 경우 설정 화면으로 이동
- [ ] 자동 로그인 실패 시 에러 메시지 표시

---

### US-C-03: 테이블 세션 라이프사이클
**As a** 매장 관리자(이수진)로서,  
**I want** 테이블 세션을 명시적으로 시작하고 종료하기를,  
**So that** 새 고객이 이전 고객의 주문 내역 없이 깨끗하게 시작할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 고객이 테이블을 이용 완료하고 자리를 떠난 상태에서
When 관리자가 이용 완료 버튼을 누르고 확인 팝업에서 확인하면
Then 현재 세션이 종료되고 주문 내역이 과거 이력으로 이동되며
     테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다
```

**추가 조건 (체크리스트)**:
- [ ] 이용 완료 버튼 클릭 시 확인 팝업 표시
- [ ] 세션 종료 후 새 고객의 첫 주문 시 새 세션 자동 시작
- [ ] 세션 종료 성공/실패 피드백 메시지 표시
- [ ] 종료된 세션의 주문은 과거 내역에서 조회 가능

---

## Epic 2: 메뉴 탐색

### US-C-04: 카테고리별 메뉴 조회
**As a** 고객(김민준)으로서,  
**I want** 카테고리 탭을 눌러 해당 카테고리의 메뉴만 보기를,  
**So that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 고객이 메뉴 화면에 있을 때
When 특정 카테고리 탭을 누르면
Then 해당 카테고리의 메뉴만 카드 형태로 표시된다
```

**추가 조건 (체크리스트)**:
- [ ] 메뉴 화면이 앱의 기본(홈) 화면으로 표시됨
- [ ] 카테고리 탭이 상단에 가로 스크롤 형태로 표시됨
- [ ] 현재 선택된 카테고리 탭이 시각적으로 강조됨
- [ ] 터치 친화적 버튼 크기 적용 (최소 44x44px)

---

### US-C-05: 메뉴 상세 정보 확인
**As a** 고객(김민준)으로서,  
**I want** 메뉴 카드에서 메뉴명, 가격, 설명, 이미지를 확인하기를,  
**So that** 주문 전에 메뉴에 대한 충분한 정보를 얻을 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] 각 메뉴 카드에 메뉴명, 가격, 설명, 이미지 표시
- [ ] 이미지가 없는 경우 기본 이미지(placeholder) 표시
- [ ] 가격은 원화(₩) 형식으로 표시
- [ ] 카드 레이아웃이 명확한 시각적 계층 구조를 가짐

---

## Epic 3: 장바구니

### US-C-06: 메뉴 장바구니 추가
**As a** 고객(김민준)으로서,  
**I want** 메뉴 카드에서 담기 버튼을 눌러 장바구니에 추가하기를,  
**So that** 원하는 메뉴를 모아서 한 번에 주문할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 고객이 메뉴 카드를 보고 있을 때
When 담기 버튼을 누르면
Then 해당 메뉴가 장바구니에 추가되고 장바구니 아이템 수가 증가한다
```

**추가 조건 (체크리스트)**:
- [ ] 이미 담긴 메뉴를 다시 추가하면 수량이 1 증가
- [ ] 장바구니 아이콘에 현재 아이템 수 배지 표시
- [ ] 추가 시 시각적 피드백 제공

---

### US-C-07: 장바구니 수량 조절
**As a** 고객(김민준)으로서,  
**I want** 장바구니에서 각 메뉴의 수량을 증가/감소시키기를,  
**So that** 주문 수량을 자유롭게 조정할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 고객이 장바구니 화면에 있을 때
When + 버튼을 누르면 수량이 1 증가하고, - 버튼을 누르면 수량이 1 감소한다
Then 총 금액이 실시간으로 업데이트된다
```

**추가 조건 (체크리스트)**:
- [ ] 수량이 0이 되면 해당 메뉴가 장바구니에서 자동 제거
- [ ] 수량은 최소 1, 최대 99
- [ ] 각 메뉴의 소계(단가 × 수량) 표시

---

### US-C-08: 장바구니 비우기
**As a** 고객(김민준)으로서,  
**I want** 장바구니 전체를 한 번에 비우기를,  
**So that** 처음부터 다시 메뉴를 선택할 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] 장바구니 비우기 버튼 클릭 시 확인 팝업 표시
- [ ] 확인 후 장바구니의 모든 메뉴가 제거됨
- [ ] 총 금액이 0으로 초기화됨

---

### US-C-09: 장바구니 로컬 저장 유지
**As a** 고객(김민준)으로서,  
**I want** 페이지를 새로고침해도 장바구니 내용이 유지되기를,  
**So that** 실수로 새로고침해도 담아둔 메뉴를 잃지 않는다.

**Acceptance Criteria (체크리스트)**:
- [ ] 장바구니 데이터가 localStorage에 저장됨
- [ ] 페이지 새로고침 후 장바구니 내용이 복원됨
- [ ] 서버 전송은 주문 확정 시에만 수행

---

## Epic 4: 주문

### US-C-10: 주문 최종 확인
**As a** 고객(김민준)으로서,  
**I want** 주문 전에 장바구니 내역을 최종 확인하기를,  
**So that** 실수 없이 정확한 메뉴를 주문할 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] 주문 확인 화면에 메뉴명, 수량, 단가, 총액 표시
- [ ] 주문 확정 버튼과 취소 버튼 제공
- [ ] 장바구니가 비어있으면 주문 확정 버튼 비활성화

---

### US-C-11: 주문 성공 플로우
**As a** 고객(김민준)으로서,  
**I want** 주문 성공 시 주문 번호를 확인하고 메뉴 화면으로 돌아가기를,  
**So that** 주문이 완료됐음을 확인하고 추가 주문을 이어서 할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 고객이 주문 확정 버튼을 눌렀을 때
When 서버에서 주문 처리가 성공하면
Then 주문 번호가 표시되고 5초 후 자동으로 메뉴 화면으로 이동하며
     장바구니가 자동으로 비워진다
```

**추가 조건 (체크리스트)**:
- [ ] 주문 번호가 화면에 명확히 표시됨
- [ ] 5초 카운트다운 표시 (선택사항)
- [ ] 즉시 이동 버튼 제공

---

### US-C-12: 주문 실패 처리
**As a** 고객(김민준)으로서,  
**I want** 주문 실패 시 에러 메시지를 보고 장바구니를 유지하기를,  
**So that** 다시 시도할 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] 주문 실패 시 에러 메시지 표시
- [ ] 실패 후 장바구니 내용 유지
- [ ] 재시도 가능한 상태로 복귀

---

## Epic 5: 주문 내역 조회

### US-C-13: 현재 세션 주문 목록 조회
**As a** 고객(김민준)으로서,  
**I want** 현재 테이블 세션에서 내가 주문한 목록을 보기를,  
**So that** 어떤 메뉴를 주문했는지 확인할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 고객이 주문 내역 탭을 눌렀을 때
When 주문 내역 화면이 로드되면
Then 현재 세션의 주문만 주문 시간 순으로 표시된다
```

**추가 조건 (체크리스트)**:
- [ ] 이전 세션(이전 고객)의 주문은 표시되지 않음
- [ ] 이용 완료 처리된 세션의 주문은 표시되지 않음
- [ ] 각 주문에 주문 번호, 시각, 메뉴 목록, 금액 표시

---

### US-C-14: 주문 상태 확인
**As a** 고객(김민준)으로서,  
**I want** 각 주문의 현재 상태(대기중/준비중/완료)를 확인하기를,  
**So that** 내 음식이 언제 나오는지 파악할 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] 각 주문에 상태(대기중/준비중/완료) 표시
- [ ] 상태별 시각적 구분 (색상 또는 아이콘)
- [ ] 상태가 변경되면 화면에 반영됨

---

## Epic 6: 관리자 인증

### US-A-01: 매장 로그인
**As a** 매장 관리자(이수진)로서,  
**I want** 매장 식별자, 사용자명, 비밀번호로 로그인하기를,  
**So that** 내 매장의 관리 시스템에 안전하게 접근할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 관리자가 로그인 화면에서 올바른 자격증명을 입력했을 때
When 로그인 버튼을 누르면
Then JWT 토큰이 발급되고 주문 모니터링 대시보드로 이동한다
```

**추가 조건 (체크리스트)**:
- [ ] 잘못된 자격증명 입력 시 에러 메시지 표시
- [ ] 새로고침 후에도 로그인 상태 유지
- [ ] 16시간 경과 후 자동 로그아웃

---

### US-A-02: 세션 유지 및 자동 로그아웃
**As a** 매장 관리자(이수진)로서,  
**I want** 16시간 동안 로그인 상태가 유지되기를,  
**So that** 영업 시간 동안 재로그인 없이 업무를 볼 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] JWT 토큰 기반으로 16시간 세션 유지
- [ ] 브라우저 새로고침 후에도 세션 유지
- [ ] 16시간 만료 후 자동으로 로그인 화면으로 이동

---

## Epic 7: 실시간 주문 모니터링

### US-A-03: 그리드 대시보드 조회
**As a** 매장 관리자(이수진)로서,  
**I want** 테이블별 주문 현황을 그리드 카드 형태로 한눈에 보기를,  
**So that** 모든 테이블의 상황을 빠르게 파악할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 관리자가 대시보드에 접속했을 때
When 화면이 로드되면
Then 테이블별 카드가 그리드 형태로 표시되고
     각 카드에 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다
```

**추가 조건 (체크리스트)**:
- [ ] 테이블별 필터링 기능 제공
- [ ] 주문이 없는 테이블도 카드로 표시

---

### US-A-04: 신규 주문 실시간 수신 (SSE)
**As a** 매장 관리자(이수진)로서,  
**I want** 새 주문이 들어오면 2초 이내에 화면에 자동 반영되기를,  
**So that** 주문을 놓치지 않고 즉시 대응할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 관리자가 대시보드를 보고 있을 때
When 고객이 주문을 확정하면
Then 2초 이내에 해당 테이블 카드에 신규 주문이 표시되고
     시각적 강조(색상 변경, 애니메이션)가 적용된다
```

**추가 조건 (체크리스트)**:
- [ ] SSE(Server-Sent Events) 기반 실시간 통신
- [ ] 신규 주문 카드에 강조 표시 적용
- [ ] SSE 연결 끊김 시 자동 재연결

---

### US-A-05: 주문 상세 보기
**As a** 매장 관리자(이수진)로서,  
**I want** 테이블 카드를 클릭해서 전체 주문 메뉴 목록을 보기를,  
**So that** 어떤 메뉴가 주문됐는지 상세히 확인할 수 있다.

**Acceptance Criteria (체크리스트)**:
- [ ] 테이블 카드 클릭 시 상세 보기 모달/패널 표시
- [ ] 전체 주문 목록: 주문 번호, 시각, 메뉴 및 수량, 총 금액 표시
- [ ] 닫기 버튼으로 대시보드 복귀

---

### US-A-06: 주문 상태 변경
**As a** 매장 관리자(이수진)로서,  
**I want** 주문 상태를 대기중/준비중/완료로 변경하기를,

## Epic 9: 메뉴 관리

### US-A-10: 메뉴 등록
**As a** 매장 관리자(이수진)로서,  
**I want** 새 메뉴를 등록하기를,  
**So that** 매장 메뉴판을 최신 상태로 유지할 수 있다.

**Acceptance Criteria (핵심 시나리오 - Given-When-Then)**:
```
Given 관리자가 메뉴 등록 화면에서 필수 정보를 입력했을 때
When 저장 버튼을 누르면
Then 새 메뉴가 등록되고 고객 메뉴 화면에 즉시 반영된다
```

**추가 조건 (체크리스트)**:
- [ ] 필수 필드: 메뉴명, 가격, 카테고리
- [ ] 선택 필드: 설명, 이미지 URL
- [ ] 필수 필드 미입력 시 에러 표시
- [ ] 가격은 0 이상의 숫자만 허용
