# 테이블오더 서비스 - 요구사항 문서

## 1. 프로젝트 개요

### 1.1 비전
디지털 주문 시스템을 통해 고객에게는 편리한 주문 경험을, 매장 운영자에게는 효율적인 운영 환경을 제공하는 테이블오더 플랫폼

### 1.2 기술 스택 결정
| 영역 | 기술 |
|------|------|
| Backend | Java + Spring Boot |
| Frontend | React (TypeScript) |
| Database | MySQL |
| 이미지 관리 | 이미지 URL 저장 (외부 링크) |
| 배포 환경 | Docker Compose (로컬 개발) |
| 실시간 통신 | Server-Sent Events (SSE) |
| 인증 | JWT (JSON Web Token) |

---

## 2. 사용자 유형 (Personas)

### 2.1 고객 (Customer)
- 테이블에 앉아 태블릿/브라우저로 주문
- 별도 로그인 없이 자동 인증
- 메뉴 탐색, 장바구니, 주문, 주문 내역 조회

### 2.2 매장 관리자 (Admin)
- 매장 식별자 + 사용자명/비밀번호로 로그인
- 실시간 주문 모니터링, 테이블 관리, 메뉴 관리

---

## 3. 기능 요구사항 (MVP 범위)

### 3.1 고객용 기능

#### FR-C-01: 테이블 자동 로그인
- 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 설정 정보 로컬 저장 후 자동 로그인
- 페이지 새로고침 시에도 세션 유지

#### FR-C-02: 메뉴 조회
- 카테고리별 메뉴 분류 및 표시
- 메뉴 카드: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 로컬 저장 (새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

#### FR-C-04: 주문 생성
- 주문 내역 최종 확인 후 확정
- 주문 성공: 주문 번호 표시 (5초) → 장바구니 비우기 → 메뉴 화면 리다이렉트
- 주문 실패: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장/테이블 식별 정보, 메뉴 목록(메뉴명/수량/단가), 총 금액, 세션 ID

#### FR-C-05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)
- 완료 처리된 세션 주문 제외

### 3.2 관리자용 기능

#### FR-A-01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 기반 인증, 16시간 세션 유지
- 비밀번호 bcrypt 해싱
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃

#### FR-A-02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드
- 각 카드: 테이블 번호, 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중/준비중/완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A-03: 테이블 관리
- 테이블 초기 설정: 번호, 비밀번호, 16시간 세션 생성
- 주문 삭제: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- 테이블 이용 완료: 확인 팝업 → 세션 종료 → 주문 이력 이동 → 현재 주문/금액 리셋
- 과거 주문 내역 조회: 테이블별, 시간 역순, 날짜 필터링

#### FR-A-04: 메뉴 관리 (MVP 포함)
- 메뉴 CRUD (등록/수정/삭제/조회)
- 메뉴 필드: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 및 가격 범위 검증

## 4. 비기능 요구사항

### 4.1 성능
- SSE 실시간 주문 표시: 2초 이내
- 메뉴 조회 응답: 1초 이내

### 4.2 보안
- JWT 토큰 기반 관리자 인증
- 비밀번호 bcrypt 해싱
- 테이블 비밀번호 검증
- 민감 정보 환경 변수 관리

### 4.3 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 레이아웃
- 명확한 시각적 계층 구조

## 5. 데이터 모델 (개요)

| 엔티티 | 주요 필드 |
|--------|-----------|
| Store | id, identifier, name, adminUsername, adminPasswordHash |
| Table | id, storeId, tableNumber, tablePassword, currentSessionId |
| TableSession | id, tableId, startedAt, completedAt |
| Menu | id, storeId, categoryId, name, price, description, imageUrl, displayOrder |
| MenuCategory | id, storeId, name, displayOrder |
| Order | id, tableId, sessionId, totalAmount, status, createdAt |
| OrderItem | id, orderId, menuId, menuName, quantity, unitPrice |

## 6. MVP 범위 요약

포함:
- 고객: 자동 로그인, 메뉴 조회, 장바구니, 주문 생성, 주문 내역 조회
- 관리자: 매장 인증, 실시간 주문 모니터링(SSE), 테이블 관리, 메뉴 관리(CRUD)

제외 (향후):
- 결제 연동
- 주방 디스플레이 시스템 (KDS)
- 다국어 지원
- 통계/분석 대시보드
