# Business Rules - Frontend

---

## 인증 규칙

### BR-AUTH-01: 테이블 자동 로그인
- localStorage에 `tableAuthConfig` 키로 저장된 설정이 있으면 자동 로그인 시도
- 자동 로그인 실패 시 초기 설정 화면으로 이동
- 토큰은 `tableAuthToken` 키로 localStorage에 저장

### BR-AUTH-02: 관리자 세션 만료
- JWT 만료 시각(`expiresAt`)을 localStorage에 저장
- 모든 API 요청 전 만료 여부 확인
- 만료된 경우 자동으로 `/admin/login`으로 리다이렉트
- 401 응답 수신 시에도 동일하게 처리

### BR-AUTH-03: 라우트 보호
- `/admin/*` 경로: 관리자 JWT 필요, 없으면 `/admin/login`으로 리다이렉트
- `/menu`, `/cart`, `/orders` 경로: 테이블 토큰 필요, 없으면 초기 설정 화면으로 이동

## 장바구니 규칙

### BR-CART-01: 중복 메뉴 처리
- 이미 담긴 메뉴를 추가하면 수량 +1 (새 항목 추가 아님)

### BR-CART-02: 수량 범위
- 최소 수량: 1 (0이 되면 항목 자동 제거)
- 최대 수량: 99

### BR-CART-03: localStorage 동기화
- 장바구니 상태 변경 시 즉시 localStorage에 저장 (`cartItems` 키)
- 앱 초기화 시 localStorage에서 복원
- 주문 성공 후 localStorage 장바구니 데이터 삭제

### BR-CART-04: 총액 계산
- `totalAmount = sum(item.price * item.quantity)`
- 수량 변경 시 실시간 재계산

## 주문 규칙

### BR-ORDER-01: 주문 확정 조건
- 장바구니가 비어있으면 주문 확정 버튼 비활성화

### BR-ORDER-02: 주문 성공 플로우
- 주문 성공 응답 수신 → 주문 번호 표시
- 5초 카운트다운 후 `/menu`로 자동 이동
- 카운트다운 중 "바로 이동" 버튼으로 즉시 이동 가능
- 이동 전 장바구니 자동 비우기

### BR-ORDER-03: 주문 실패 처리
- API 오류 시 에러 모달 표시 (중요 에러)
- 장바구니 내용 유지
- 재시도 가능 상태 유지

## UI/UX 규칙

### BR-UI-01: 장바구니 사이드 드로어
- 메뉴 화면에서 장바구니 아이콘/버튼 클릭 시 오른쪽에서 슬라이드
- 드로어 외부 클릭 또는 닫기 버튼으로 닫힘
- 드로어 열린 상태에서도 메뉴 화면 스크롤 가능

### BR-UI-02: 에러 처리 분류
- **모달 표시**: 주문 실패, 세션 만료, 네트워크 오류
- **Toast 표시**: 주문 상태 변경 성공, 메뉴 저장 성공, 일반 API 오류
- Toast 표시 시간: 3초

### BR-UI-03: 신규 주문 강조 (관리자)
- SSE로 `ORDER_CREATED` 이벤트 수신 시 해당 테이블 카드에 강조 스타일 적용
- 관리자가 해당 카드를 클릭(상세 보기)하면 강조 해제
- 강조 상태는 `SseContext`의 `newOrderTableIds` Set으로 관리

### BR-UI-04: 반응형 그리드 (관리자 대시보드)
- 화면 너비 < 768px: 1열
- 768px ~ 1024px: 2열
- 1024px ~ 1440px: 3열
- 1440px 이상: 4열

### BR-UI-05: 터치 친화적 버튼
- 모든 인터랙티브 요소 최소 크기: 44x44px

## 데이터 검증 규칙 (관리자 메뉴 관리)

### BR-VALID-01: 메뉴 등록/수정 폼
- 메뉴명: 필수, 최대 100자
- 가격: 필수, 0 이상의 정수
- 카테고리: 필수 선택
- 설명: 선택, 최대 500자
- 이미지 URL: 선택, URL 형식 검증
